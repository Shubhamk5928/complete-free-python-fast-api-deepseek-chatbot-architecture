Executive Summary: A Free, Customizable, Code-Based Chatbot

This document outlines a comprehensive, cost-effective, and highly customizable plan for implementing a full-stack chatbot for the early-stage startup, invisiblebrowser.in. The proposed solution is tailored for a backend engineer who prefers a code-based approach over off-the-shelf, no-code platforms.

The entire system is designed to be implemented using free, open-source technologies, eliminating upfront software costs and providing maximum flexibility for future development and scaling. The architecture is modular, separating concerns between the user interface, backend logic, AI processing, and external communication, which facilitates maintainability and independent scaling of components.

Core Technology Stack

FastAPI Backend Framework
DeepSeek R1-V2 AI Model
MinionAgent Framework
Telegram Bot API

Key Innovation

The core of the system leverages a powerful, free AI model for generating intelligent responses based on real-time web and site data. A key feature is its intelligent user engagement tracking, which conditionally forwards chat logs to the business owner via a preferred messaging platform after a set number of interactions, enabling proactive lead capture without requiring user registration.

The system is built with multilingual capabilities at its core, ensuring a broad and inclusive user experience from the outset. This plan provides not only the high-level architectural design but also detailed, step-by-step implementation guidance, including specific libraries, code structures, and integration patterns to be used within the existing backend codebase.

Frontend: Web-Based Chat Interface

The frontend will be a custom-built, lightweight component that can be easily embedded into any page of the invisiblebrowser.in website. It will be developed using standard web technologies: HTML for structure, CSS for styling, and JavaScript for interactivity.

Key Features:

  • • Clean, intuitive, and responsive design
  • • Asynchronous JavaScript (AJAX) calls
  • • Real-time message display without page reload
  • • Customizable styling to match existing branding

Backend: Python Web Service

The backend will serve as the core of the chatbot system, acting as the central hub that orchestrates all interactions between the frontend, the AI engine, and the external communication services. FastAPI is recommended due to its high performance and native support for asynchronous operations.

Core Responsibilities:

  • • Authenticating and managing user sessions
  • • Tracking message counts per session
  • • Forwarding queries to AI engine
  • • Triggering conditional forwarding logic

AI Engine: DeepSeek R1-V2 with MinionAgent

The AI Engine is the brain of the chatbot, responsible for understanding user queries and generating intelligent, context-aware responses. This will be achieved by combining the power of the DeepSeek R1-V2 large language model with the autonomous capabilities of the MinionAgent framework.

Key Capabilities:

Core Implementation: Answering User Questions

The foundational capability of the proposed chatbot is its ability to intelligently answer user questions by synthesizing information from both the public web and the specific content of invisiblebrowser.in.

Recommended Free Model: DeepSeek R1-V2

For an early-stage startup operating with limited capital, selecting a high-performance, cost-effective AI model is paramount. The recommended choice is DeepSeek R1-V2, an open-weight large language model developed by DeepSeek AI[131].

Why DeepSeek R1-V2?

  • • Free to use - No licensing fees or per-token charges
  • • Strong reasoning capabilities - Excels at complex, multi-step tasks
  • • Multilingual support - Native proficiency in English and Chinese
  • • Efficient architecture - Lower hardware requirements

Unlike proprietary models from providers like OpenAI or Anthropic, which incur per-token costs that can quickly escalate with user traffic, DeepSeek's open-weight models can be run locally or on low-cost cloud infrastructure, eliminating variable API expenses[131].

Model Specifications

Type: Open-weight LLM
Cost: Free
Languages: English, Chinese +
Architecture: Efficient design

Web Scraping with MinionAgent

To fulfill the requirement of answering questions based on both the public web and the invisiblebrowser.in website, the chatbot needs a dynamic data retrieval mechanism. The recommended tool is MinionAgent, a powerful, open-source AI agent framework designed for browser automation and information retrieval[133].

Installation

pip install minion-agent-x

The MinionAgent framework is available as a Python package, making it easy to integrate into the existing backend codebase[135].

MinionAgent acts as the "hands" of the LLM, allowing it to interact with the live web. When a user asks a question, the chatbot can instruct MinionAgent to navigate to relevant websites, search for information, and extract the necessary data.

Agent Configuration

Integration Example

Python 
import asyncio
from minion_agent import MinionAgent, AgentConfig, AgentFramework
from dotenv import load_dotenv

load_dotenv()

async def handle_user_query(user_query: str):
    agent_config = AgentConfig(
        model_id="deepseek-r1-v2",
        name="web_researcher",
        description="An agent that can browse the web to answer questions.",
        tools=["minion_agent.tools.browser_tool.browser"],
    )
    
    agent = await MinionAgent.create(AgentFramework.SMOLAGENTS, agent_config)
    scraped_data = await agent.run(user_query)
    
    # Pre-process scraped_data and create LLM prompt
    prompt = f"Based on the following information, answer '{user_query}':\n\n{scraped_data}"
    
    return llm_response

# Called from chat API endpoint

Alternative Implementation: RAG with LlamaIndex

While the MinionAgent approach provides dynamic web scraping, an alternative architectural pattern known as Retrieval-Augmented Generation (RAG) offers different trade-offs. A RAG system pre-processes and indexes specific documents into a vector database for faster, more reliable answers.

RAG Pipeline Components

  • • Ollama - Local LLM serving framework[130]
  • • LlamaIndex - Data framework for RAG applications
  • • Vector Database - Efficient similarity search
  • • Document Processing - Website content indexing

Ollama Setup

Bash 
# Install and run DeepSeek model
ollama run deepseek-r1:1.5b

RAG Implementation

Python 
from llama_index.core import VectorStoreIndex, SimpleDirectoryReader
from llama_index.llms.ollama import Ollama

# Load and index documents
documents = SimpleDirectoryReader("data/").load_data()
index = VectorStoreIndex.from_documents(documents)

# Create query engine
llm = Ollama(model="deepseek-r1:1.5b")
query_engine = index.as_query_engine(llm=llm)

# Query the index
response = query_engine.query(
    "What are the features of invisiblebrowser.in?"
)

Implementation: Conditional Chat Forwarding

A key feature of the proposed chatbot is its ability to intelligently manage user interactions based on their engagement level and authentication status. After an unauthenticated user has engaged in four chat exchanges, the system will automatically forward the entire conversation log to the startup team via a messaging platform.

User Session and State Management

To implement the conditional forwarding feature, the backend must track the state of each user's interaction with the chatbot. This involves monitoring whether a user is logged in and counting the number of messages they have sent in a single session.

Session State Tracking

Check authentication token or session cookie

Message Counter

Increment with each message from unauthenticated users

Session Timeout

User Interface Internationalization (i18n)

To provide a truly global user experience, the chatbot's user interface (UI) must be as multilingual as its core AI engine. This involves implementing internationalization (i18n) using dedicated libraries to manage and display UI elements in the user's preferred language.

Flask-Babel Implementation

For a Flask-based backend, Flask-Babel is the most widely adopted library for internationalization. It provides comprehensive tools for managing translations, formatting dates/numbers, and handling pluralization[101].

pip install flask-babel

The core principle involves replacing all hardcoded strings with special translation functions (typically _()), which act as markers for strings that should be translated based on the user's selected locale.

2 Extract to .pot template
3 Create .po translation files
4 Compile to .mo binary files

Implementation Example

Python/Flask 
@app.before_request
def before_request():
    # Check for language parameter
    if 'lang' in request.args:
        session['lang'] = request.args['lang']
        babel.refresh()
    
    # Check session for language
    lang = session.get('lang', None)
    if lang:
        g.lang = lang
    else:
        # Auto-detect from browser
        g.lang = request.accept_languages.best_match(['en', 'fr', 'de', 'zh'])

Backend Integration: Routes, Controllers, and Services

The backend will expose a clean and intuitive RESTful API to handle all chatbot-related operations. This section details the architectural organization of routes, controllers, and services that form the core business logic of the application.

API Route Design

POST /chat/message

Primary Endpoint

Core endpoint for receiving user messages. Frontend sends POST request with user message in JSON body.

{ "message": "Hello! I have a question..." }

GET /chat/history

Secondary Endpoint

Retrieves conversation history for current session. Returns JSON array of message objects.

[
  {"sender": "user", "text": "Hello!"},
  {"sender": "bot", "text": "How can I help?"}
]

Request Flow

1 Frontend sends user message
2 Controller validates request
3 ChatManager processes message
4 AIService generates response
5 Return response to frontend
Additional Endpoints
  • • GET /chat/status - Check session status
  • • POST /chat/language - Set language preference
  • • GET /health - Service health check

Controller Layer (ChatController)

The controller layer acts as the intermediary between the API routes and the service layer. It handles incoming HTTP requests, validates request data, calls appropriate service methods, and formats responses to be sent back to the frontend.

Controller Responsibilities

Handle incoming HTTP requests
Validate request data and parameters
Orchestrate service method calls
Format and return HTTP responses
Handle errors and exceptions

ChatController Example

Python 
class ChatController:
    def __init__(self, chat_manager):
        self.chat_manager = chat_manager
    
    def send_message(self, request_data):
        """Handle incoming chat message"""
        try:
            # Validate request
            message = request_data.get('message')
            if not message:
                return error_response("Message required", 400)
            
            # Process through chat manager
            response = self.chat_manager.process_message(message)
            
            return success_response(response)
            
        except Exception as e:
            return error_response(str(e), 500)
    
    def get_history(self, session_id):
        """Retrieve chat history"""
        try:
            history = self.chat_manager.get_session_history(session_id)
            return success_response(history)
        except Exception as e:
            return error_response(str(e), 500)

Separation of Concerns

Controller:

Handles "what" needs to be done

Benefits
  • • Clean, maintainable code

process_message()

2. AIService

3. MessageForwarder

4. Response

Returns AI response to user via controller

@app.before_request
def before_request():
    # Check for language parameter
    if 'lang' in request.args:
        session['lang'] = request.args['lang']
        babel.refresh()
    
    # Check session for language
    lang = session.get('lang', None)

        g.lang = lang
    else:
        # Auto-detect from browser

Controller Layer (ChatController)

The controller layer acts as the intermediary between the API routes and the service layer. It handles incoming HTTP requests, validates request data, calls appropriate service methods, and formats responses to be sent back to the frontend.

Controller Responsibilities

Handle incoming HTTP requests
Validate request data and parameters
Orchestrate service method calls
Format and return HTTP responses
Handle errors and exceptions

ChatController Example

Python 
class ChatController:
    def __init__(self, chat_manager):
        self.chat_manager = chat_manager
    
    def send_message(self, request_data):
        """Handle incoming chat message"""
        try:
            # Validate request
            message = request_data.get('message')
            if not message:
                return error_response("Message required", 400)
            
            # Process through chat manager
            response = self.chat_manager.process_message(message)
            
            return success_response(response)
            
        except Exception as e:
            return error_response(str(e), 500)
    
    def get_history(self, session_id):
        """Retrieve chat history"""
        try:
            history = self.chat_manager.get_session_history(session_id)
            return success_response(history)
        except Exception as e:
            return error_response(str(e), 500)
